LLMs are better at writing code than at orchestrating tool calls. Datafetch mounts a dataset as a typed virtual filesystem — df.db.*, df.lib.*, df.answer — so the agent writes TypeScript over a real interface. Every accepted commit becomes a typed call in your tenant's lib/.

/ THE LOOP  >>>  publish · mount · shape · optimise

      

        
A dataset is an environment.
Mounted, shaped, returned.

        
Datafetch turns a dataset into a tenanted environment. The provider publishes once. Each agentic search shapes the workspace to fit the user's intent shape. Every accepted trajectory teaches the provider how the data is actually used, so serving, indexes, and the data shape itself can be tightened against real demand.
